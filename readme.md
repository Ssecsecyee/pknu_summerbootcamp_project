# 🎣 FishMaster AI: 스마트 낚시 가이드 플랫폼

> **💡 프로젝트 소개**
> 개인 낚시꾼을 위한 AI 기반 스마트 가이드 플랫폼입니다. 이미지 인식 기술과 위치 데이터 API를 결합하여 안전하고 법규를 준수하는 낚시 환경을 제공합니다.

---

## 🛠 주요 기능

### 1. 📷 이미지 기반 어종 판별
- **AI 어종 분석:** 촬영된 물고기 이미지를 분석하여 어종을 즉시 식별합니다.
- **식용 여부 확인:** 해당 어종의 식용 가능 여부 및 법적 보호종(금지어종) 여부를 즉시 안내합니다.

### 2. 📏 체장 측정 및 규제 준수 가이드
- **실시간 체장 분석:** 이미지 내 물고기의 크기를 측정합니다.
- **법적 포획 기준 비교:** 분석된 체장이 해당 어종의 법적 '금지 체장' 기준을 위반하는지 실시간으로 판단하여 경고 또는 알림을 제공합니다.

### 3. 📍 낚시 가능 구역 안내
- **위치 기반 필터링:** Google Maps API 및 Kakao Maps API를 연동합니다.
- **구역 정보 제공:** 사용자의 현재 위치를 기반으로 낚시 금지 구역 여부를 판별하여 정보를 제공합니다.

### 4. 🤖 낚시 지수 기반 AI 챗봇
- **실시간 환경 데이터 분석:** 기상 데이터, 조석 정보(물때), 풍속 등 낚시에 중요한 환경 지표를 종합 분석합니다.
- **낚시 적합도 안내:** 현재 위치와 시간에 따른 '낚시 적합 지수'를 계산하여 출조 여부를 결정하는 데 도움을 줍니다.
- **맞춤형 상담:** 사용자의 질문에 대해 현재 환경을 바탕으로 한 맞춤형 출조 조언 및 낚시 팁을 실시간으로 답변합니다.

## 📷 이미지 기반 어종 판별

### 1. 데이터 구성

### 식용 가능 어종

- 총 **49종**
- 어종별 최소 **3장 이상의 이미지**
- 이미지 유형
  - 어류 전체 모습이 선명한 사진
  - 물속에서 촬영된 사진
  - 실제 낚시로 포획된 사진
  - 다양한 방향과 배경에서 촬영된 사진

### 식용 불가 어종

- 총 **3종**
  - 국매리복
  - 기름치
  - 별복
- 어종별 최소 **3장 이상의 이미지**
- 각 어종의 식용 불가 사유를 별도 JSON 데이터로 관리

> 어종별 3장은 시스템 동작 확인을 위한 최소 데이터입니다. 실제 AI 학습 정확도를 높이기 위해서는 어종별 최소 50장 이상, 가능하면 100장 이상의 이미지가 필요합니다.

---

## 2. YOLO 적용 방식

본 프로젝트에서는 YOLO를 이용해 사진 속 어류 영역을 먼저 검출합니다.

```text
원본 이미지 입력
        ↓
YOLO 객체 탐지
        ↓
어류 영역 검출
        ↓
Bounding Box 기준 이미지 자르기
        ↓
배경 제거 또는 크기 조정
        ↓
어종 분류 모델 입력
        ↓
어종명 및 식용 가능 여부 출력
```

사진에 어류가 한 마리만 존재하더라도 배경, 사람 손, 낚싯대, 바닥 등이 함께 포함될 수 있으므로, YOLO로 어류 영역을 먼저 검출한 뒤 어종을 분류하는 방식으로 구성합니다.

## 3. 이미지 수집 및 정리

수집한 이미지를 어종별 폴더로 구분합니다.

```text
raw_dataset/
├── 가리비/
├── 갈치/
├── 감성돔/
├── 고등어/
├── 국매리복/
├── 기름치/
├── 별복/
└── ...
```

각 이미지의 파일명은 다음과 같이 통일합니다.

```text
감성돔_001.jpg
감성돔_002.jpg
감성돔_003.jpg
```

이미지 수집 시 다음 조건을 고려합니다.

- 물고기 전체 형태가 잘 보이는 사진
- 물속에서 촬영된 사진
- 실제 낚시로 포획된 사진
- 정면, 측면, 위쪽 등 다양한 촬영 각도
- 밝은 환경과 어두운 환경
- 바닥, 선박, 쿨러 등 다양한 배경
- 손, 낚싯대, 뜰채 등이 함께 등장하는 실제 환경 사진

수집된 이미지는 어종명이 정확한지 먼저 확인한 후 폴더에 저장합니다. 동일한 이미지가 반복되어 포함되지 않도록 중복 파일도 함께 검사합니다.


### 2. 📏 체장 측정 및 규제 준수 가이드/상세 프로세스

이 기능은 컴퓨터 비전 기술과 법적 규제 데이터를 결합하여, 사용자가 올바르고 안전한 낚시 문화를 실천하도록 돕는 자동화 가이드입니다.

### ① 이미지 기반 데이터 전처리
* **사용자 입력:** 어종이 포함된 이미지 업로드.
* **참조 스케일 생성:** 이미지 내 물고기의 크기를 상대적으로 측정하기 위해 픽셀 단위의 데이터를 물리적 길이(cm)로 변환하는 비례식 계산 과정을 거칩니다.

### ② 실시간 체장 및 어종 분석
* **체장 추출:** 탐지된 물고기의 경계 상자(Bounding Box) 및 핵심 특징점 데이터를 기반으로 실제 체장을 산출합니다.

### ③ 법적 기준 대조 및 가이드 제공
* **데이터베이스 연동:** 식별된 어종별로 해양수산부의 법적 '금지 체장' 데이터를 MongoDB에서 실시간으로 조회합니다.
* **규제 위반 판정:** 측정된 체장이 금지 체장 기준보다 작을 경우, 즉각적인 경고 메시지를 사용자에게 송출하여 방류를 유도합니다.
* **결과 안내:** 분석된 어종의 명칭, 측정된 체장, 법적 기준 준수 여부를 포함한 최종 리포트를 출력합니다.

## 2. 방안 비교

| 구분 | **방안 A: 상용 API 연동 (카카오모빌리티 등)** | **방안 B: 자체 라우팅 엔진 구축 (OSM + OSRM)** |
| :--- | :--- | :--- |
| **핵심 원리** | 외부 서버에 경로를 요청하여 결과값 수신 | 서버 내 자체 라우팅 엔진을 직접 운영 |

## 3. 세부 실행 계획

### [방안 A] 상용 API 연동 방식
1. **API 키 확보**: 카카오모빌리티 등 서비스별 디벨로퍼스 계정 생성 및 API 키 발급.
2. **데이터 호출**: 출발지/도착지 위경도 좌표를 API 파라미터로 전송.
3. **결과 처리**: 응답받은 JSON 데이터(경로 좌표 리스트)를 지도 서비스에 렌더링.
4. **기대 효과**: 최소한의 개발 리소스로 안정적인 경로 탐색 서비스 조기 런칭.

### [방안 B] 자체 라우팅 엔진 구축 방식
1. **데이터 확보**: OSM(OpenStreetMap) 데이터 및 공공 교통 데이터(GTFS, ITS 등) 수집.
2. **엔진 구축**: Docker 기반 OSRM 혹은 GraphHopper 엔진 서버 세팅.
3. **전처리 파이프라인**: 수집된 데이터를 엔진 규격에 맞게 변환 및 인덱싱.
4. **가중치 로직 적용**: 실시간 교통 데이터를 연동하여 엔진 내 링크 가중치 동적 업데이트.
5. **기대 효과**: 외부 의존성 제거 및 서비스 정책에 맞춘 유연한 경로 최적화 로직 적용.

## 🔄 2. AI Agent Workflow (AI 에이전트 워크플로우)

> **🧠 AI 에이전트 핵심 아키텍처**
> 사용자의 요청(텍스트/이미지)이 입력되면 **EXAONE LLM**이 의도를 분석하고 분류합니다. 이후 검증 단계를 거쳐 최적의 기능 도구(`Tool`)를 호출하고, 최종적으로 데이터베이스 조회 결과와 AI 모델 추론 결과를 결합하여 직관적인 자연어 답변을 도출합니다.

### 📌 에이전트 작동 프로세스

1. **사용자 요청 수신 (`User Input`):** 사용자가 채팅창을 통해 질문을 입력하거나 물고기 사진을 업로드합니다.
2. **의도 분석 (`Intent Classification`):** `EXAONE LLM`이 사용자의 질문 맥락을 분석하여 어떤 작업(어종 분류, 낚시 규제 조회, 위험 구역 확인, 일반 대화)이 필요한지 판별합니다.
3. **데이터 검증 (`Schema Validation`):** 정의된 Pydantic 스키마를 통해 AI 에이전트의 매개변수가 누락 없이 올바르게 매핑되었는지 검증합니다.
4. **기능 도구 호출 (`Tool Execution`):** 판별된 카테고리에 맞는 전용 도구(`Tool Layer`)를 호출하여 실행합니다.
   - **데이터 저장소 및 모델 연동:** 각 도구는 `MongoDB`에서 관련 지리/규제 데이터를 쿼리하거나, 업로드된 이미지를 `YOLO` 모델로 전달해 실시간 객체 탐지를 수행합니다.
5. **자연어 답변 생성 (`NLG`):** 도구 실행 결과 반환된 정형 데이터(JSON)를 `EXAONE LLM`이 다시 받아 사용자가 이해하기 쉬운 자연스러운 문장 및 가이드 리포트로 재구성하여 최종 전달합니다.

## 🛠 3. 주요 기능 및 구현 상세

### 1️⃣ 📷 이미지 기반 어종 판별 (YOLO + 어종 분류 모델)

> **🔍 핵심 기능 정의**
> 단일 이미지 내에서도 배경, 인물 손, 낚싯대, 바닥 등 다양한 요소가 포함되기 때문에 **객체 탐지(YOLO)**로 어류 영역을 먼저 분할(Crop)하여 불필요한 배경 노이즈를 제거한 뒤 **어종 분류 모델**의 추론 정확도를 극대화합니다.

#### 📌 YOLO 객체 탐지 및 어종 분류 파이프라인
1. **원본 이미지 입력:** 사용자가 업로드한 낚시 결과물 사진을 시스템에 등록합니다.
2. **YOLO 객체 탐지 (`Object Detection`):** 딥러닝 기반 탐지 엔진이 이미지 내에서 '어류(Fish)' 객체의 정확한 위치를 추출합니다.
3. **어류 영역 검출 및 자르기 (`Localization & Crop`):** 검출된 바운딩 박스(Bounding Box) 좌표를 기준으로 어류 부분만 자릅니다.
4. **전처리 및 크기 조정 (`Preprocessing`):** 잘린 이미지에서 노이즈를 제어하고, 분류 모델 규격에 맞게 리사이징을 거칩니다.
5. **어종 분류 모델 추론 (`Classification`):** 정제된 이미지를 입력받아 CNN 기반 분류 엔진이 정확한 어종 명칭을 식별합니다.
6. **최종 리포트 출력:** 식별된 최종 어종명과 데이터베이스 연동 정보(식용 가능 여부 등)를 매칭하여 반환합니다.

```text
원본 이미지 입력 ➔ YOLO 객체 탐지 ➔ 어류 영역 검출 ➔ Bounding Box 이미지 자르기 ➔ 배경 제거/조정 ➔ 어종 분류 모델 입력 ➔ 최종 결과 도출
```

#### 📌 데이터셋 구성 및 가이드라인
* 🐠 **식용 가능 어종 (49종):** 어종별 최소 3장 이상의 이미지 데이터를 확보합니다.
* ⚠️ **식용 불가 어종 (3종):** `국매리복`, `기름치`, `별복`으로 정의하며, 각 어종의 구체적인 식용 불가 사유는 독립적인 JSON 포맷의 메타데이터로 연결합니다.
* 📸 **수집 검증 기준:** 
  - 물속 렌더링, 실제 낚시 포획, 정면/측면/상단 다각도 촬영물 위주로 확보합니다.
  - 바닥, 선박, 아이스박스, 사람 손, 뜰채 등 실제 현장 노이즈가 복합적으로 구성된 이미지를 통해 실전 정확도를 개선합니다.
  *(※ 프로토타입 단계에서는 동작 확인을 위해 어종당 최소 3장으로 구성하나, 실서빙을 위한 모델 최적화 단계에서는 50~100장 이상의 데이터 보강이 필요합니다.)*

### 2️⃣ 📏 체장 측정 및 규제 준수 가이드

> **⚖️ 핵심 기능 정의**
> 컴퓨터 비전 기술과 공공 데이터베이스의 수산법 규제 데이터를 결합하여, 사용자가 올바르고 안전한 방류 문화를 실천하도록 돕는 자동화 가이드라인입니다.

#### 📌 체장 측정 및 비교 가이드 프로세스
1. **참조 스케일 및 비례식 계산 (`Pixel to Metric`):** 이미지 내 어체와 대비되는 기준 객체 또는 픽셀 정밀 데이터 분석을 통해 실제 물리 길이(cm) 단위로 변환하기 위한 변환 비례 관계를 생성합니다.
2. **체장 중심점 추출 (`Length Measurement`):** YOLO가 추출한 좌표 및 어체의 머리 끝(Head)에서 꼬리 끝(Tail)까지의 핵심 특징점을 기하학적으로 연산하여 실측 체장을 측정합니다.
3. **수산자원 규제 데이터 실시간 대조 (`DB Validation`):** 식별된 어종명을 인덱싱하여 `MongoDB`에 적재된 해양수산부 수산자원관리법상의 '포획 금지 체장' 데이터를 즉시 쿼리합니다.
4. **금지 체장 위반 판정 및 경고 (`Compliance Check`):** 분석된 실측 체장이 규정 크기 미만일 경우, 법적 처벌 위험성 안내 및 방류를 유도하는 팝업 UI를 노출합니다.
5. **피드백 리포트 생성:** 측정 길이, 기준 체장 정보, 법적 준수 적격 여부가 한눈에 포함된 카드 형태의 요약 리포트를 출력합니다.

### 4️⃣ 🤖 낚시 지수 기반 AI 챗봇

> **🌤️ 핵심 기능 정의**
> 단순 텍스트 질의응답을 넘어 기상 및 바다 조석 상태를 반영하여, 해당 지역의 조과를 예측하고 맞춤형 낚시 전술 조언을 제공하는 Context-Aware 인공지능 비서 서비스입니다.

#### 📌 주요 챗봇 연동 설계
1. **실시간 기후·환경 메트릭스 수집:** 기상청 및 국립해양조사원 API를 백그라운드로 연결하여 풍속, 파고, 수온, 물때(조석 예보) 등의 최신 포인트 데이터를 획득합니다.
2. **지능형 적합 지수 산출 (`Activity Index`):** 수집된 다차원 환경 데이터를 결합하여 오늘 목적지의 '종합 낚시 적합 지수'를 내부 모델을 통해 정량적으로 평가하고 시각적 인디케이터로 제공합니다.
3. **컨텍스트 맞춤형 피드백 제공 (`Contextual NLG`):** `EXAONE LLM`에 기후 정량 메트릭을 시스템 프롬프트 컨텍스트로 전달하여, 사용자의 질의에 따라 가장 전문적이고 직관적인 맞춤형 팁(낚시 기법, 주의사항 등)을 대화로 풀어서 제공합니다.
