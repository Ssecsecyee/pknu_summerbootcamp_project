# 🎣 FishMaster AI: 스마트 낚시 가이드 플랫폼

개인 낚시꾼을 위한 AI 기반 스마트 가이드 플랫폼입니다. 이미지 인식 기술과 위치 데이터 API를 결합하여 안전하고 법규를 준수하는 낚시 환경을 제공합니다.

---

## 🛠 주요 기능

### 1. 📷 이미지 기반 어종 판별
- **AI 어종 분석:** 촬영된 물고기 이미지를 분석하여 어종을 즉시 식별합니다.
- **식용 여부 확인:** 해당 어종의 식용 가능 여부 및 법적 보호종(금지어종) 여부를 즉시 안내합니다.

### 2. 📏 체장 측정 및 규제 준수 가이드
- **실시간 체장 분석:** 이미지 내 물고기의 크기를 측정합니다.
- **법적 포획 기준 비교:** 분석된 체장이 해당 어종의 법적 '금지 체장' 기준을 위반하는지 실시간으로 판단하여 경고 또는 알림을 제공합니다.

### 3. 📍 낚시 가능 구역 안내
- **위치 기반 필터링:** Google Maps API 및 Kakao Maps API를 연동합니다.
- **구역 정보 제공:** 사용자의 현재 위치를 기반으로 낚시 금지 구역 여부를 판별하여 정보를 제공합니다.

### 4. 🤖 낚시 지수 기반 AI 챗봇
- **실시간 환경 데이터 분석:** 기상 데이터, 조석 정보(물때), 풍속 등 낚시에 중요한 환경 지표를 종합 분석합니다.
- **낚시 적합도 안내:** 현재 위치와 시간에 따른 '낚시 적합 지수'를 계산하여 출조 여부를 결정하는 데 도움을 줍니다.
- **맞춤형 상담:** 사용자의 질문에 대해 현재 환경을 바탕으로 한 맞춤형 출조 조언 및 낚시 팁을 실시간으로 답변합니다.

## 2. 데이터 구성

### 식용 가능 어종

- 총 **49종**
- 어종별 최소 **3장 이상의 이미지**
- 이미지 유형
  - 어류 전체 모습이 선명한 사진
  - 물속에서 촬영된 사진
  - 실제 낚시로 포획된 사진
  - 다양한 방향과 배경에서 촬영된 사진

### 식용 불가 어종

- 총 **3종**
  - 국매리복
  - 기름치
  - 별복
- 어종별 최소 **3장 이상의 이미지**
- 각 어종의 식용 불가 사유를 별도 JSON 데이터로 관리

> 어종별 3장은 시스템 동작 확인을 위한 최소 데이터입니다. 실제 AI 학습 정확도를 높이기 위해서는 어종별 최소 50장 이상, 가능하면 100장 이상의 이미지가 필요합니다.

## 3. YOLO 적용 방식

본 프로젝트에서는 YOLO를 이용해 사진 속 어류 영역을 먼저 검출합니다.

```text
원본 이미지 입력
        ↓
YOLO 객체 탐지
        ↓
어류 영역 검출
        ↓
Bounding Box 기준 이미지 자르기
        ↓
배경 제거 또는 크기 조정
        ↓
어종 분류 모델 입력
        ↓
어종명 및 식용 가능 여부 출력
```

사진에 어류가 한 마리만 존재하더라도 배경, 사람 손, 낚싯대, 바닥 등이 함께 포함될 수 있으므로, YOLO로 어류 영역을 먼저 검출한 뒤 어종을 분류하는 방식으로 구성합니다.

## 4.1 이미지 수집 및 정리

수집한 이미지를 어종별 폴더로 구분합니다.

```text
raw_dataset/
├── 가리비/
├── 갈치/
├── 감성돔/
├── 고등어/
├── 국매리복/
├── 기름치/
├── 별복/
└── ...
```

각 이미지의 파일명은 다음과 같이 통일합니다.

```text
감성돔_001.jpg
감성돔_002.jpg
감성돔_003.jpg
```

이미지 수집 시 다음 조건을 고려합니다.

- 물고기 전체 형태가 잘 보이는 사진
- 물속에서 촬영된 사진
- 실제 낚시로 포획된 사진
- 정면, 측면, 위쪽 등 다양한 촬영 각도
- 밝은 환경과 어두운 환경
- 바닥, 선박, 쿨러 등 다양한 배경
- 손, 낚싯대, 뜰채 등이 함께 등장하는 실제 환경 사진

수집된 이미지는 어종명이 정확한지 먼저 확인한 후 폴더에 저장합니다. 동일한 이미지가 반복되어 포함되지 않도록 중복 파일도 함께 검사합니다.


### 2. 📏 체장 측정 및 규제 준수 가이드/상세 프로세스

이 기능은 컴퓨터 비전 기술과 법적 규제 데이터를 결합하여, 사용자가 올바르고 안전한 낚시 문화를 실천하도록 돕는 자동화 가이드입니다.

### ① 이미지 기반 데이터 전처리
* **사용자 입력:** 어종이 포함된 이미지 업로드.
* **참조 스케일 생성:** 이미지 내 물고기의 크기를 상대적으로 측정하기 위해 픽셀 단위의 데이터를 물리적 길이(cm)로 변환하는 비례식 계산 과정을 거칩니다.

### ② 실시간 체장 및 어종 분석
* **체장 추출:** 탐지된 물고기의 경계 상자(Bounding Box) 및 핵심 특징점 데이터를 기반으로 실제 체장을 산출합니다.

### ③ 법적 기준 대조 및 가이드 제공
* **데이터베이스 연동:** 식별된 어종별로 해양수산부의 법적 '금지 체장' 데이터를 MongoDB에서 실시간으로 조회합니다.
* **규제 위반 판정:** 측정된 체장이 금지 체장 기준보다 작을 경우, 즉각적인 경고 메시지를 사용자에게 송출하여 방류를 유도합니다.
* **결과 안내:** 분석된 어종의 명칭, 측정된 체장, 법적 기준 준수 여부를 포함한 최종 리포트를 출력합니다.

## 1. 개요
본 프로젝트의 목적은 출발지와 목적지 간의 최단 경로 및 소요 시간을 산출하는 시스템을 구현하는 것임. 이를 위해 외부 상용 API 연동 방식과 자체 라우팅 엔진 구축 방식을 비교 검토하여 최적의 방안을 결정하고자 함.

---

## 2. 방안 비교

| 구분 | **방안 A: 상용 API 연동 (카카오모빌리티 등)** | **방안 B: 자체 라우팅 엔진 구축 (OSM + OSRM)** |
| :--- | :--- | :--- |
| **핵심 원리** | 외부 서버에 경로를 요청하여 결과값 수신 | 서버 내 자체 라우팅 엔진을 직접 운영 |
| **장점** | 개발 속도 빠름, 데이터 정확도 및 신뢰도 높음 | 트래픽 비용 절감, 커스터마이징 자유도 높음 |
| **단점** | 호출량에 따른 비용 발생, 로직 수정 불가 | 데이터 전처리 및 서버 운영/유지보수 필요 |
| **적합한 상황** | 서비스 초기, 정밀한 실시간 교통 정보 중요 | 대규모 트래픽 발생, 고도화된 특수 로직 필요 |

---

## 3. 세부 실행 계획

### [방안 A] 상용 API 연동 방식
1. **API 키 확보**: 카카오모빌리티 등 서비스별 디벨로퍼스 계정 생성 및 API 키 발급.
2. **데이터 호출**: 출발지/도착지 위경도 좌표를 API 파라미터로 전송.
3. **결과 처리**: 응답받은 JSON 데이터(경로 좌표 리스트)를 지도 서비스에 렌더링.
4. **기대 효과**: 최소한의 개발 리소스로 안정적인 경로 탐색 서비스 조기 런칭.

### [방안 B] 자체 라우팅 엔진 구축 방식
1. **데이터 확보**: OSM(OpenStreetMap) 데이터 및 공공 교통 데이터(GTFS, ITS 등) 수집.
2. **엔진 구축**: Docker 기반 OSRM 혹은 GraphHopper 엔진 서버 세팅.
3. **전처리 파이프라인**: 수집된 데이터를 엔진 규격에 맞게 변환 및 인덱싱.
4. **가중치 로직 적용**: 실시간 교통 데이터를 연동하여 엔진 내 링크 가중치 동적 업데이트.
5. **기대 효과**: 외부 의존성 제거 및 서비스 정책에 맞춘 유연한 경로 최적화 로직 적용.

---

## 4. 종합 제안 및 결론
* **제안**: **방안 A(API 연동)**를 통해 프로토타입을 신속히 구현하여 핵심 기능을 검증할 것을 권장함.
* **이유**: 개발 초기 단계에서는 라우팅 엔진의 복잡한 운영 리스크보다 사용자 경험(UX) 확보가 우선임. 추후 트래픽 증가로 인한 비용 효율성 문제나 서비스 고도화 요구가 발생할 경우 **방안 B**로 단계적 전환을 검토하는 것이 효율적임.
